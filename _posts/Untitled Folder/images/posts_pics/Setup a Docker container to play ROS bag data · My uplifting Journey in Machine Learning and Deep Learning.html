Setup a Docker container to play ROS bag data

  • docker

  • ros


28 Oct 2016

For the first challenges of the Self-Driving Car Nanodegree (SDCND), Udacity Team released a big bunch of driving data. The data is stored in ROS bags, and required the [ROS](http://wwww.ROS.org) package. My first attempt to install ROS directly on my MacBook (see the specs on the right) was a total failure: during the install, I got a lot of warnings related to missing dependencies. In the course of fixing those, I ended up altering a few other packages that I had previously installed. A few people on SDC slack Channel adviced to use [Docker](https://www.docker.com/) and to run ROS in a container. In this post, I will explain how I setup a docker image on Mac OS in order to not only open ROS bag files but also play the videos from the container. The container runs Ubuntu 14.04 and ROS, with a few more packages. Here are the specs of my machine (not much of a powerhouse):
  • Macbook Pro 13"
  • Processor 2.7 GHz Intel Core i5
  • Memory 8Gb
  • MacOS Sierra V.10.12
  • Graphics Intel Iris Graphics 6100 1536 Mb
The rosbag file is saved on an external Volume. I will save all dataset in there. The rest is kept on the primary Hard Drive

There are 4 main steps:

  1. Install Docker

  2. Create an Image (filesystem and parameters to use at runtime) with the required packages: OS, ROS Indigo etc…

  3. Run a container (instance of an image)

  4. Run a python script to play the images in the rosbag file

1. Install Docker for Mac

Go to webpage of Docker and, follow the instructions detailed on the website.

Below are a few commands to remember (we will use them later)

  • List all images: docker ps -a

  • Delete all containers: docker rm $(docker ps -a -q)

  • Delete all images docker rmi $(docker images -q)

2. Create an image

On Docker webpage, there is a detailed tutorial on how to build an image. However, we will be focusing on making an image to meet our need, i.e the image will have: + Ubuntu 14.04 as the OS, + ROS Indigo (to vizualize and extract data from ros bags), and + some more tools and packages (python, scikit-image, ….).

At first, we create a new folder where we store all the files that are needed to create the image and run a container. Note that the rosbag file does not have to be in that same folder: in my case, I store the dataset in an external hard drive).

  • in ../Documents/, we create the folder sdc_docker: $ mkdir sdc_docker

  • Move to new directory: cd sdc_docker

  • In ../Documents/sdc_docker/, we create a file Dockerfile. The file will contain all the commands required to generate our image. touch Dockerfile

3. Run the container

In order to run an instance of our image, open a new Terminal and type the command: docker run -i -t sdc-u14 /bin/bash

"docker run" : docker runs or create a docker container an instance "-i" interactive mode "-t" attach terminal to dock sdc-u14 is the name of the image to be loaded into the container.

  • How do we get the IP adress: We run the following command in the MacOS Terminal: ` echo “show State:/Network/Global/IPv4” | scutil | grep PrimaryInterface | awk ‘{print $3}’ | xargs ifconfig | grep inet | grep -v inet6 | awk ‘{print $2}’ `

4. Run a python script to display the images

Hey, this is our final step…oofff…but it requires a few sub-steps :-(

4.1. Additional tools:

We need to install 2 more packages: SOCAT and XQuartz.

  • SOCAT Open a MacOS Terminal and run brew install socat

  • XQuartz : install from the dmg file available here Note that you might need to logout after the install.

  • view_rosbag_video.py : enables to play the rosbag file. get a copy here for download. !!! We will copy the file in the same location where we store the rosbag file.

  • attach-docker-container.sh: this sh file allows to attach the container to more Terminal. The file can be downloaded here. We save that file with the other sh files, in ../documents/sdc-dockers/

4.2. Set up

First we need to run socat in background. Type the following in a MacOS shell: ` socat TCP-LISTEN:6000,reuseaddr,fork UNIX-CLIENT:"$DISPLAY" `

We launch XQuartz: open -a XQuartz

Now, things start to get more interesting! We gonna open 3 XQuartz Terminals (Applications>Terminals), and follow the instructions from Marius S. here. Basically, it involves having 3 XQuartz Terminals attached to the same image.

  • XQuartz Terminal1: go to the folder ../Documents/sdc-docker/ and to get in the container, run : sh docker-run.sh

  • XQuartz Terminal2: Make sure to be in the right directory: /Documents/src-dockers/, and then run: sh attach-container.sh <id_value> Then, Terminal2 should be also in the container. Now run those 2 lines:

source /opt/ros/indigo/setup.bash roscore

  • XQuartz Terminal3: Make sure to be in the right directory: /Documents/src-dockers/, and then run: sh attach-container.sh <id_value>

We now go to the directory where we store the dataset.bag file, and run :

rosbag play -s 120 dataset.bag

After 6 minutes, data from the rosbag starts to be printed out

  • Go back to XQuartz Terminal1: Move to the folder where the dataset is stored, and run: python view_rosbag_video.py

And, voila!
